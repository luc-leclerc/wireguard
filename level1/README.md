# WireGuard - Level 1 - The basics

Setting up a very basic WireGuard with auto-generate keys for the server and one client.

# Pre-requisite to test this project

* docker, docker-compose are installed
* Admin access to your router
* An extra device (ex.: a smartphone)

# How to run this setup and confirm that it works

* Run `./start.sh`
1. Configure your public facing router (Warning: Each router brand/model have their own set of features and UI; use a search engine for help, good luck!)
    * Enable port forwarding of UDP port `LEVEL1_PUBLIC_FACING_LISTENING_PORT` to the server that will run WireGuard, at port `$LEVEL1_DOCKER_CONTAINER_LISTENING_PORT`.
2. Start the server
    * Run `./start.sh` to start the server, if all goes well, a QR code will be shown in the console.
3. Test that it works with a client device (ex.: your smartphone)
    * First, disable wifi on it and check that your have a different public IP than your home network.
    * Install the WireGuard client on it, and click add, then scan the QR Code generated by the server.
    * Enable the VPN, and then check again what is public IP, it should now be the same as your home network. Congrats!

# Potential issues

* This setup hardcode the VPN on network `10.42.130.0/24`, it might conflict with your pre-existing networks.
* This setup hardcode network interface to `eth0`, yours might have a different name.
